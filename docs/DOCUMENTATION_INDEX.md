## What was removed

The docs folder previously contained many dated implementation summaries, parallel agent reports, duplicate setup/deployment specs, and troubleshooting one-offs. Those files were removed because they were not canonical, were likely to go stale, and increased cognitive load for future agents/developers.

Going forward: update the closest canonical doc instead of creating a new markdown file unless the new file has a clear long-term owner and purpose.
